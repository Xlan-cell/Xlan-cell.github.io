4.关系模型

关系模型基本概念

1.关系

关系模型中的表格称为“关系”，是一个二维表的形式，每个关系由行和列组成。行代表实体，列代表属性。

2.属性

二维表的列称为属性，每个属性都有一个属性名.根据不同的习惯也可以称为字段

3.元组（Tuple）：

关系中的每一行称为一个元组，也就是一条记录。每个元组由多个属性值组成，对应一个具体的数据项。

4.域（Domain）：

每个属性都有一个取值范围或集合，称为域。域定义了属性可能的取值，例如，年龄属性的域可能是所有正整数。

6.Sql的类别

数据定义语言

数据查询语言（DQL - Data Query Language）

  • 主要语句：SELECT
  • 功能：用于从数据库中查询数据。DQL 主要用于数据检索，而不是数据的增删改。通过 SELECT 语句，可以从表中检索特定的行和列，并进行筛选、排序、分组等操作。

数据定义语言（DDL - Data Definition Language）

  • 主要语句：CREATE、ALTER、DROP、TRUNCATE
  • 功能：用于定义和修改数据库结构，包括创建、修改或删除数据库、表和视图等对象。例如，CREATE TABLE 用于创建表，ALTER TABLE 用于修改表结构，DROP TABLE 用于删除表。

数据操作语言（DML - Data Manipulation Language）

  • 主要语句：INSERT、UPDATE、DELETE
  • 功能：用于对表中的数据进行操作。DML 用于插入、更新和删除数据行。例如，INSERT INTO 用于添加新数据，UPDATE 用于修改现有数据，DELETE 用于删除数据。

数据控制语言（DCL - Data Control Language）

  • 主要语句：GRANT、REVOKE
  • 功能：用于管理数据库的访问权限，控制用户对数据库的操作权限。例如，GRANT 用于赋予用户权限，REVOKE 用于撤销权限。DCL 主要用于数据库的安全控制。
