1.依赖导入 认识knife4i

knife4i 是一个针对 Spring Boot 和 Spring Cloud 的增强型文档生成工具，主要用于生成和管理 RESTful API 文档。它是对 Swagger 的增强版，提供了更多功能和更好的用户体验。下面详细介绍 knife4i 的主要功能和如何在 Spring Boot 项目中使用它。

主要功能
API 文档生成：自动生成 RESTful API 文档。
UI 增强：提供更加友好的 UI 界面，方便查看和调试 API。
接口测试：直接在浏览器中测试接口。
代码生成：支持多种语言的客户端 SDK 代码生成。
接口管理：方便管理和维护 API 文档。

3.注解认识

@api

用于标记接口类或接口类的方法，描述接口的作用。

@ApiOperation

作用：用于标记接口类的方法，描述具体操作的作用。
示例：

 @ApiOperation(value = "获取用户列表", notes = "返回所有用户的列表")
 @GetMapping("/users")
 public List<User> getUsers() {
     // ...
 }
@ApiModel

作用：用于描述模型对象，通常用于实体类

 @ApiModel(description = "用户信息")
 public class User {
     @ApiModelProperty(value = "用户ID")
     private Long id;

     @ApiModelProperty(value = "用户名")
     private String name;

     // 其他字段和方法
 }
@ApiModelProperty

作用：用于描述模型对象中的属性。

 @ApiModelProperty(value = "用户ID", required = true)
 private Long id;
@ApiParam

作用：用于描述参数。

 @GetMapping("/user/{id}")
 public User getUser(@ApiParam(value = "用户ID", required = true) @PathVariable Long id) {
     // ...
 }
@ApiResponses

作用：用于描述方法的响应结果。
示例：

 @ApiOperation(value = "获取用户列表")
 @ApiResponses({
     @ApiResponse(code = 200, message = "成功"),
     @ApiResponse(code = 404, message = "未找到")
 })
 @GetMapping("/users")
 public List<User> getUsers() {
     // ...
 }
@ApiResponse

作用：用于描述单个响应结果。
示例：

 @ApiResponse(code = 200, message = "成功", response = User.class)
@ApiImplicitParams
 @ApiOperation(value = "获取用户列表")
 @ApiImplicitParams({
     @ApiImplicitParam(name = "page", value = "页码", paramType = "query", dataType = "int"),
     @ApiImplicitParam(name = "size", value = "每页数量", paramType = "query", dataType = "int")
 })
 @GetMapping("/users")
 public List<User> getUsers(@RequestParam int page, @RequestParam int size) {
     // ...
 }
@ApiImplicitParam

作用：用于描述单个隐式参数。
示例：

 @ApiImplicitParam(name = "page", value = "页码", paramType = "query", dataType = "int")
@ApiIgnore

作用：用于忽略某个类或方法，使其不被文档化。
示例：

  @ApiIgnore
  @GetMapping("/health")
  public String healthCheck() {
      return "OK";
  }
